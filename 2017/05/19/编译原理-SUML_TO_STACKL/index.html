C++实现SUML到STACKL微型简易编译器

发表于 2017-05-19 | 分类于 计算机基础知识

问题分析

在学习中科大华保健老师《编译原理》在线视频课程时，遇到了这样一道课后练习题。C语言版本的代码实现，详见我的上一篇博客。

动手实现一个简易的编译器，把加法表达式（语言）便以为栈式命令（语言）.
例如：4+5+6 转换为 PUSH 4, PUSH 5, ADD, PUSH 6, ADD

其实看似简单的一个问题，其中涉及了很多编译器相关的理论。正好借此回顾整理一下。编译器的核心人物是将源程序代码翻译为目标代码。那么，我们可以吧4+5+6看作是加法表达式语言（假定为一种源语言）。PUSH 4, PUSH 5, ADD, PUSH 6, ADD这些命令的集合看作是目标代码。

拿到源语言代码的字符流4+5+6之后，第一步工作就是词法拆分，将字符流拆分为一个一个的分词。本例中则可以拆分为5个分词：

第二步工作是，构建抽象语法树进行类型检查，检查抽象语法树是否符合源语言的语法规则。加法表达式需要满足左结合律，也即((4+5) + 6)，这样的结构，很明显可以考虑通过构建二叉树来实现。

后续遍历上图的二叉树，一次吧每个节点的值存入一个栈中。需要注意的是，遇到数值则直接压入栈中，遇到算数运算符号，则从栈中出栈两个元素，做相应运算后包运算结果再次压入栈中（。。。。有木有很熟悉，跟数据结构里面，用栈实现四则运算是不是很相似呢。）。
