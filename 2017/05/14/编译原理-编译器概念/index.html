编译原理笔记（一）：编译器基础概念

发表于 2017-05-14 | 分类于 计算机基础知识

编译器概述

我们通常编程所用的语言是高级语言，中间需要经过编译器的编译，翻译为计算机能够执行的机器语言（二进制数据），才能在计算机上执行。这里所说的计算机既指物理计算机（PC，台式机等物理设备），也指虚拟计算机（比如Java虚拟机JVM）。

编译器本质上是一个软件程序，其核心功能是把源代码翻译成目标代码。这里的源代码具体指的是C，C++，Java，SQL等高级语言代码，而目标代码针对不同的计算机体系结构，指的是X86，IA64，ARM，MIPS等。

需要注意编译器与解释器的区别：解释器也是一类处理程序的程序。编译器负责吧源代码翻译为目标代码，不负责去执行目标代码得到相应的计算结果，而解释器的职责恰恰就是解释目标代码并得到代码的执行结果。

编译器结构

编译器具有非常模块化的高层结构，可以看成是由多个阶段构成的流水线结构。

一种更复杂的编译器结构。

总结起来就是，编译器有多个不同的阶段（前端，后端）组成，每个阶段都要使用不同的理论、数据结构和算法来处理不同的问题。

编译器实例

实现一个检疫编译器，把一个加法表达式语言（1+2+3）编译成一个栈式计算机（比如，我们非常熟悉的JVM其实就是一个栈式计算机）。

分析后发现，我们要实现的栈式计算机需要：

  • 一个或多个操作数栈
  • 两条指令：压栈指令（push n），加法指令（add）

阶段一 词法语法分析

词法分析，也就是分析上述加法表达式的有哪些单词构成，具体可以拆分为3个操作数和2个加法运算符，共5个部分。语法分析，就是判断这5个部分是否符合加法表达式语言的语法规则，也就是要满足加法运算的左结合律。

阶段二 构建语法树

学习过数据结构你应该很快就明白了，不光是加法计算需要用到二叉树，其实四则运算用到的逆波兰，实质上也是通过后续遍历二叉树进行求值计算的。根据加法的计算规则，构建的语法树如下。
