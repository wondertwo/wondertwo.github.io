GitLab 配置windows公钥和私钥（ssh-key）

发表于 2017-08-17 | 分类于 GIT

windows 本地 Git 配置

  1. 可通过--global 参数配置全局的用户名和邮箱
  1. 也可省略--global 参数，在项目目录下配置本项目的用户名和邮箱

配置 windows 本地 ssh-key

  1. 如果你是windows，并且安装了tortoiseGit，那在tortoiseGit安装目录下，比如 C:\Program Files\TortoiseGit\bin，会有一个产生密钥对的工具 puttygen.exe 。打开 puttygen.exe ，点击Generate，注意，需要鼠标在进度条下的空白区域不停的晃动，它会根据鼠标位置来随机产生密钥对。

  1. 复制上面的ssh-rsa公钥到gitlab下，选择add ssh key，粘贴进去。

  1. 保存私钥save private key , 一般保存在C:\Users\Administrator\下的一个目录里，我放在了.ssh下，重命名为 id_rsa.ppk，如果没有此目录可以建一个其他的目录。同时你也可以将上述公钥也保存一份在相同的文件夹下，重命名为id_rsa.pub。
  2. 打开本地安装的tortoiseGit设置界面（鼠标右键tortoiseGit选择setting），选择Git->Remote->putty选择上一步保存的ppk文件。

  1. 设置ssh客户端：tortoiseGit-> setting->network->ssh client->TortoiseGitPlink.exe。

  1. 在项目git pull和 git pull 以及 git push 的时候，勾选autoload putty key，(如果上一步的ssh 客户端选错可能导致此处为灰色无法勾选)。

  1. 试一下push和pull功能，应该就需要再输入密码了。
